# W2 AI Assistant

AI-ассистент для постановки задач на разработку: менеджер описывает фичу свободным текстом, модель ведёт себя как аналитик — задаёт уточняющие вопросы, подсвечивает пробелы в описании и формирует итоговую задачу по настраиваемому шаблону.

## Описание проекта

Инструмент сокращает разрыв между «идеей в голове» и понятным для команды ТЗ. Пользователь вводит черновик фичи; ассистент на базе **Claude API** помогает довести формулировку до состояния, пригодного для бэклога или тикета: собирает контекст, выявляет неясности и выдаёт структурированный результат в заданном формате.

**Стек:** React, Claude API (Anthropic).

## Ключевые функции

- **Свободное описание фичи** — ввод текста без жёстких полей на старте.
- **Уточняющие вопросы** — диалог в стиле аналитика, чтобы закрыть неоднозначности.
- **Подсветка пропусков** — явное указание, чего не хватает в описании (цели, границы, критерии и т.д.).
- **Генерация задачи по шаблону** — итоговый текст собирается по настраиваемому шаблону (например, под ваш Jira/YouTrack/внутренний формат).

## Как запустить

> Репозиторий в ранней стадии: после появления `package.json` и исходников шаги могут отличаться — обновите этот раздел под фактические скрипты.

1. **Требования:** Node.js LTS, менеджер пакетов (`npm`, `pnpm` или `yarn`).
2. **Клонирование и зависимости:**
   ```bash
   git clone <url-репозитория>
   cd w2-ai-assistent
   npm install
   ```
3. **Переменные окружения:** создайте файл `.env` (или `.env.local`) и укажите ключ к Claude API, например:
   ```env
   VITE_ANTHROPIC_API_KEY=sk-ant-...
   ```
   Имя переменной зависит от того, как настроен фронтенд (часто для Vite используют префикс `VITE_`). **Не коммитьте** файлы с секретами.
4. **Запуск в режиме разработки:**
   ```bash
   npm run dev
   ```
5. Откройте в браузере адрес, который выведет dev-сервер (обычно `http://localhost:5173` для Vite).

Если ключ API вызывается только с сервера (рекомендуется для продакшена), добавьте прокси или отдельный backend и перенесите ключ туда — в README тогда опишите два процесса: `npm run dev` для фронта и запуск API.
